Get the network id string from the single result provided.

This contains profile ids for all components and their percentages for the matched IP range, delimited by ':'. The profile id percentage pairs are concatenated with the separator character '|'.

Parameters
results - pointer to the results
result - pointer to the result to get the network id of
destination - pointer to the memory to write the characters to
size - amount of memory allocated to destination
componentProfileIndex - the combination profile index to start constructing the network ID from. Negative values indicate nothing to fetch.
exception - pointer to an exception data structure to be used if an exception occurs. See exceptions.h
Returns
the combination profile index to start from in the next call for fetching the remaining of the network ID. All indices will be -1s there is no more to fetch. The returned format does not contain leading or trailing separator so the separator will need to be added explicitly if more than one call is required.

◆ fiftyoneDegreesIpiBalancedConfig

fiftyoneDegreesConfigIpi fiftyoneDegreesIpiBalancedConfig

Uses caching to balance memory usage and performance.

A connection is maintained to the source data file to load data into caches when required. As the cache is loaded, memory will increase until the cache capacity is reached. The concurrency of each collection must be set to the maximum number of concurrent operations to optimize file reads. This is the default configuration.

◆ fiftyoneDegreesIpiDefaultConfig

fiftyoneDegreesConfigIpi fiftyoneDegreesIpiDefaultConfig

Default detection configuration.

This configures the data set to not create a temp file.

◆ fiftyoneDegreesIpiHighPerformanceConfig

fiftyoneDegreesConfigIpi fiftyoneDegreesIpiHighPerformanceConfig

Highest performance configuration.

Loads all the data into memory and does not maintain a connection to the source data file used to build the data set. The concurrency setting is ignored as there are no critical sections with this configuration.

◆ fiftyoneDegreesIpiInMemoryConfig

fiftyoneDegreesConfigIpi fiftyoneDegreesIpiInMemoryConfig

IP INTELLIGENCE CONFIGURATIONS.

Configuration to be used where the data set is being created using a buffer in memory and concepts like caching are not required. The concurrency setting is ignored as there are no critical sections with this configuration.

◆ fiftyoneDegreesIpiLowMemoryConfig

fiftyoneDegreesConfigIpi fiftyoneDegreesIpiLowMemoryConfig

Low memory configuration.

A connection is maintained to the source data file used to build the data set and used to load data into memory when required. No caching is used resulting in the lowest memory footprint at the expense of performance. The concurrency of each collection must be set to the maximum number of concurrent operations to optimize file reads.

◆ fiftyoneDegreesIpiSingleLoadedConfig

fiftyoneDegreesConfigIpi fiftyoneDegreesIpiSingleLoadedConfig

Configuration designed only for testing.

This uses a loaded size of 1 in all collections to ensure all every get and release calls can be tested for items which do not exist in the root collection. This configuration is not exposed through C++ intentionally as it is only used in testing.
